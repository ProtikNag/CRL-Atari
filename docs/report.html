CRL: Continual RL Consolidation Report

Protik Nag  |  April 2026  |  Continual RL Atari Classic Control DQN

1. Overview

This report summarizes experiments on consolidating independently-trained DQN experts into a single multi-task network, evaluated across two domains: Atari (Breakout, SpaceInvaders, Pong) and Classic Control (CartPole, Acrobot, LunarLander). We compare our primary method, Weighted Hessian Consolidation (WHC), against knowledge distillation (Distillation, Hybrid), sequential continual learning baselines (EWC, TRAC, C-CHAIN, Progress & Compress), and a multi-task oracle.

On Atari, Hybrid achieves the highest consolidation retention (48.5% average), driven by near-expert Pong performance (90.2%). Multi-task joint training sets the oracle ceiling at 53.4%. WHC retains 26.2% on SpaceInvaders and 0.9% on Breakout but fails on Pong. On Classic Control, no single method dominates all three tasks: WHC achieves 97.3% CartPole retention, TRAC reaches 100%, but both fail Acrobot entirely. C-CHAIN and Multi-task are the only methods to partially solve Acrobot.

2. Experimental Setup

Atari (Image-Based)
  • Architecture: 3-layer CNN (32-64-64 channels, kernels 8-4-3, strides 4-2-1) with 512-unit FC hidden layer and a unified 6-action output head covering the union action space
  • Algorithm: Double DQN, 3M timesteps per game, learning rate 1e-4, batch size 32
  • Replay buffer: 500K transitions, AdamW optimizer, gradient clipping at 10.0
  • Exploration: Linear epsilon decay from 1.0 to 0.01 over 500K steps
  • Tasks: Breakout, SpaceInvaders, Pong (union action space = 6 actions)
Classic Control (Vector-State)
  • Architecture: 2-layer MLP (128-128 hidden units), unified 4-action output head
  • Algorithm: Double DQN, 200K timesteps per task, learning rate 1e-3, batch size 64
  • State representation: Observations zero-padded to ℝ8 (max dimension across tasks)
  • Tasks: CartPole-v1 (dim 4, 2 actions), Acrobot-v1 (dim 6, 3 actions), LunarLander-v3 (dim 8, 4 actions)

Evaluation Protocol

  • Atari: 30 episodes per game; Classic Control: 500 episodes per task
  • Deterministic policy (argmax over valid actions), action masking via −∞
  • Seed 42 throughout; all methods evaluated from the same expert checkpoints

3. Consolidation Methods

A. Weighted Hessian Consolidation (WHC) Eq. 11–13, Primary

Closed-form solution using a surrogate loss formed by Hessian-weighted quadratic approximations expanded at each expert's own optimum. Placing expansions at the true optima eliminates the gradient term and minimizes the Taylor remainder.

Hagg = Σi αi Hi   |   bagg = Σi αi Hi wi*
ŵλ = (Hagg + λI)−1 (bagg + λw̄)
  • Fisher computed at each expert's own checkpoint (not a shared anchor)
  • Uniform task weights αi = 1/N by default
  • Fisher samples: 20,000 high-confidence states per task
  • Single closed-form solution (no iterative optimization)

B. Knowledge Distillation Baseline

Trains a student starting from the ensemble mean to match the soft Q-value distributions of all experts using temperature-scaled KL divergence. Q-values are normalized per-task before softmax.

LKD = (1/N) Σi T2 DKL(σ(Qexpert/T) || σ(Qstudent/T))
  • 10,000 epochs, lr = 5e-5, temperature T = 0.01, batch size 64
  • Per-task Q-value normalization (zero-mean, unit-variance) before softmax

C. Hybrid (Iterative Taylor + KD) Algorithm 2

Two-phase approach. Phase 1 provides a Taylor warm start via multi-round joint consolidation; Phase 2 refines via knowledge distillation from the warm-start solution.

  • Phase 1: K = 10 rounds of Taylor updates, η0 = 0.9, γ = 0.9
  • Phase 2: 10,000 KD epochs, lr = 5e-5, T = 0.01

Baselines

D. Elastic Weight Consolidation (EWC) Kirkpatrick et al. 2017

Sequential continual learning with a Fisher-weighted L2 penalty anchoring parameters toward previous task solutions.

LEWC(θ) = Ltask(θ) + (λ/2) Σi,j Fij (θj − θi,j*)2
  • λ = 5000, Fisher samples: 5000 per task, Online EWC decay γ = 0.95
  • Sequential training on task order; penalty accumulates across tasks

E. Multi-Task Joint Training Oracle Upper Bound

Single DQN trained simultaneously on all tasks via round-robin environment stepping and random task sampling. Not subject to catastrophic forgetting; serves as an oracle upper bound. Atari: 6M total steps; Classic Control: 600K total steps.

F. TRAC Classic Control Only

Adaptive continual learning with robust tracking. Combines Adam optimization with an adaptive trust region that automatically controls step sizes to prevent catastrophic forgetting during sequential task training. Parameter-free (no λ to tune).

G. C-CHAIN Classic Control Only

Churn-aware continual learning. Manages policy churn by penalizing large changes in Q-value predictions on previously seen states during sequential training. Addresses the gap between optimization-level forgetting and behavioral-level policy instability.

H. Progress & Compress (P&C) Schwarz et al. 2018

Dual-network architecture: a "progress" (active) column learns each new task, then the learned knowledge is compressed into a "knowledge base" network via distillation after each task. The knowledge base provides lateral connections to the active column.

6. Cross-Domain Analysis

WHC: Strengths and Limitations

WHC's closed-form solution (Eq. 9) provides a principled, optimization-free method for consolidating expert models. Its performance profile is consistent across domains: strong retention on the simplest task (CartPole 97.3%, SpaceInvaders 26.2%) but catastrophic failure on tasks requiring fine-grained temporal control (Pong −21.0, Acrobot −500).

The fundamental bottleneck is the diagonal Fisher approximation. The true Hessian of a policy network has significant off-diagonal structure encoding how parameters interact across layers and timesteps. The diagonal approximation captures per-parameter importance but discards these interactions, producing a consolidated model that preserves individual parameter magnitudes but not their coordinated behavior. This explains why WHC succeeds on CartPole (where near-linear control suffices) but fails on Pong (where precise paddle-ball tracking requires coordinated feature activations).

KD-Based Methods vs. Hessian Methods

On Atari, Distillation and Hybrid both achieve ~90% Pong retention, dramatically outperforming WHC and EWC. KD operates directly on the Q-value output distribution rather than the parameter space, allowing it to preserve the functional behavior of each expert without relying on accurate curvature estimation. The Hybrid method's Taylor warm-start provides a modest advantage on SpaceInvaders (52.8% vs 36.1%) but does not improve Pong retention (90.2% vs 90.3%).

On Classic Control, however, the KD advantage largely disappears. Distillation achieves only 23.5% CartPole retention (vs WHC's 97.3%) and fails Acrobot entirely. The Classic Control domain's smaller networks and lower-dimensional state spaces may not provide enough capacity for KD to find a good shared representation, or the lower temperature may be over-sharpening the target distributions for simpler policies.

Sequential Methods and Task-Order Sensitivity

EWC, TRAC, C-CHAIN, and P&C all train sequentially on the task order (CartPole → Acrobot → LunarLander). The results reveal strong task-order effects:

  • TRAC perfectly retains CartPole (Task 1) but completely loses Acrobot (Task 2). Its adaptive trust region protects early tasks at the expense of later ones.
  • C-CHAIN trades CartPole retention (1.9%) for Acrobot (69.4% normalized) and LunarLander (−39.2), showing the most balanced sequential transfer.
  • P&C achieves the only positive LunarLander score (53.2) among all methods, suggesting its compress phase effectively distills useful representations from earlier tasks into the knowledge base.
  • EWC on Atari retains the most Breakout (10.5%) among consolidation methods but fails on Pong (Task 3), consistent with the asymmetric protection bias of sequential Fisher penalties.

Multi-Task as Oracle

Multi-task joint training sets the ceiling on Atari at 53.4% average retention, with near-perfect SpaceInvaders (96.3%) but weaker Pong (28.5%) and Breakout (35.5%). The Pong gap is notable: Multi-task scores only 5.7 while both Distillation and Hybrid exceed 18.0. This suggests that round-robin environment stepping does not allocate sufficient learning signal to Pong's sparse reward structure, whereas KD can directly transfer the expert's converged policy.

On Classic Control, Multi-task is the only method to substantively solve Acrobot (−121.6) while maintaining moderate CartPole (154.6) and LunarLander (−63.6). Its LunarLander score remains negative, indicating that even simultaneous training struggles with the landing precision required by this task.

7. Next Steps

  • Block-diagonal Fisher for WHC: Replace the diagonal Fisher with a K-FAC or block-diagonal approximation to capture cross-parameter interactions. This is the highest-priority improvement given WHC's consistent failure on temporally demanding tasks.
  • Task-weighted initialization: Anchor the WHC regularization term toward the expert of the hardest task (Pong/Acrobot) rather than the uniform ensemble mean, to reduce dilution of task-specific features.
  • C-CHAIN + WHC hybrid: C-CHAIN's balanced sequential transfer could serve as initialization for WHC, combining behavioral preservation with Hessian-weighted consolidation.
  • KD early stopping: Both Distillation and Hybrid may benefit from validation-based stopping criteria, as the distillation epoch sweep showed non-monotone performance on some tasks.
  • Scaling to more tasks: With N = 3, ensemble-mean dilution is moderate. Hierarchical merging strategies (pairwise WHC followed by full consolidation) should be explored for N ≥ 5.
  • LunarLander investigation: Only P&C achieves positive LunarLander scores. Understanding why its compress phase succeeds where other methods fail could inform future consolidation designs.
