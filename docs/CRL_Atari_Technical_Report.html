1. Overview and Motivation

This project implements a Continual Reinforcement Learning (CRL) framework for Atari games. The core challenge is training a single model that performs well across multiple Atari games without catastrophic forgetting, where learning a new task degrades performance on previously learned tasks.

Approach

The pipeline follows a train experts, then consolidate paradigm:

  1. Expert Training — Train separate DQN agents, one per Atari game (Breakout, SpaceInvaders, Pong). All experts share a union action space (the union of each game’s minimal actions, computed at runtime), with per-game action masking.
  2. Consolidation — Merge the expert models into a single global model using one of three methods:
    • Elastic Weight Consolidation (EWC) — Fisher-weighted parameter regularization
    • Knowledge Distillation — Temperature-scaled soft-target matching
    • HTCL — Second-order Taylor expansion with closed-form parameter updates
  3. Evaluation — Compare the consolidated models against individual experts on all tasks to measure knowledge retention and forward/backward transfer.

Key Design Decisions

  • Union action space — The output head covers the union of each game’s minimal action set (6 actions for Pong/Breakout/SpaceInvaders). Invalid actions are masked to −∞. This allows direct weight sharing and merging.
  • Independent expert initialization — Each expert is initialized from random weights, avoiding any coupling between the sequential training order.
  • PopArt normalization — Q-value output layer weights are rescaled on-the-fly to handle scale differences across games (Pong rewards in [−1, 1] vs. SpaceInvaders in [0, 500+]).
  • Diagonal Fisher — Used by both EWC and HTCL as a computationally tractable approximation to the Hessian, enabling element-wise closed-form updates in HTCL.
Reference: Nag, P., Raghavan, K., Narayanan, V. (2026). “Mitigating Task-Order Sensitivity and Forgetting via Hierarchical Second-Order Consolidation.” ICML 2026. arXiv:2602.02568.

3. DQN Network Architecture

The DQN uses a standard convolutional neural network (CNN) backbone followed by fully connected layers. The key architectural choice is the union action space output head that covers the union of minimal action sets across all games.

Architecture

Input:  (batch, 4, 84, 84)    # 4 stacked grayscale frames, 84×84 pixels

Layer 1: Conv2d(4 → 32,  kernel=8×8, stride=4, padding=0) + ReLU
         Output: (batch, 32, 20, 20)

Layer 2: Conv2d(32 → 64, kernel=4×4, stride=2, padding=0) + ReLU
         Output: (batch, 64, 9, 9)

Layer 3: Conv2d(64 → 64, kernel=3×3, stride=1, padding=0) + ReLU
         Output: (batch, 64, 7, 7)

Flatten: (batch, 64 × 7 × 7) = (batch, 3136)

FC:      Linear(3136 → 512) + ReLU

Output:  Linear(512 → 6)   # Q-values for union action space

Total parameters: ~1.7 million

Dueling DQN (optional)

Instead of a single output head, splits into two streams:

  • Value stream: Linear(512 → 256) → Linear(256 → 1)
  • Advantage stream: Linear(512 → 256) → Linear(256 → 6)
Q(s, a) = V(s) + A(s, a) − meana' A(s, a')

Why a Union Action Space?

Rather than using all 18 ALE joystick actions, we compute the union of each game’s minimal action set at runtime. For Pong + Breakout + SpaceInvaders this yields 6 actions:

GameMinimal ActionsALE Indices
Pong6 actions{0, 1, 3, 4, 11, 12}
Breakout4 actions{0, 1, 3, 4}
SpaceInvaders6 actions{0, 1, 3, 4, 11, 12}

Union = {0, 1, 3, 4, 11, 12} = NOOP, FIRE, RIGHT, LEFT, RIGHTFIRE, LEFTFIRE (6 actions). This is a much smaller head than the full 18-action space, resulting in fewer parameters and a more efficient representation. Invalid actions per game are still masked to −∞.

4. Union Action Space and Action Masking

The Arcade Learning Environment (ALE) defines 18 possible actions for the Atari joystick:

IndexActionIndexActionIndexAction
0NOOP6UPRIGHT12LEFTFIRE
1FIRE7UPLEFT13DOWNFIRE
2UP8DOWNRIGHT14UPRIGHTFIRE
3RIGHT9DOWNLEFT15UPLEFTFIRE
4LEFT10UPFIRE16DOWNRIGHTFIRE
5DOWN11RIGHTFIRE17DOWNLEFTFIRE

Action Masking

During action selection, the agent computes Q(s, a) for all 6 union actions, then applies:

mask[a] = 0 if a ∈ valid_actions, −∞ otherwise

Qmasked(s, a) = Q(s, a) + mask[a]

a* = argmaxa Qmasked(s, a)

This ensures the agent never selects an invalid action while maintaining a shared weight structure.

Action Mapping (UnionActionWrapper)

The ALE environment expects local action indices (0 to n_game_actions − 1), not union indices. The UnionActionWrapper translates:

Example for Pong (minimal actions = [0, 1, 3, 4, 11, 12]):
    Union = [0, 1, 3, 4, 11, 12]  (all 6 are valid for Pong)
    union index 0 (NOOP)       → local index 0
    union index 1 (FIRE)       → local index 1
    union index 2 (RIGHT)      → local index 2
    union index 3 (LEFT)       → local index 3
    union index 4 (RIGHTFIRE)  → local index 4
    union index 5 (LEFTFIRE)   → local index 5

Example for Breakout (minimal actions = [0, 1, 3, 4]):
    union index 0 (NOOP)       → local index 0
    union index 1 (FIRE)       → local index 1
    union index 2 (RIGHT)      → local index 2
    union index 3 (LEFT)       → local index 3
    union index 4 (RIGHTFIRE)  → INVALID (masked to −∞)
    union index 5 (LEFTFIRE)   → INVALID (masked to −∞)

This wrapper is the outermost wrapper (applied last, after all other wrappers) so that inner wrappers such as FireResetEnv still operate on the game’s local action space. The agent issues union indices, which the wrapper maps to local indices before passing them down.

Epsilon-Greedy Exploration

During training, with probability ε, the agent selects a random action uniformly from the valid action set (not all 6 union actions):

a ~ Uniform(valid_actions) with probability ε
a = argmax Qmasked(s, a) with probability 1 − ε

Epsilon decays linearly from 1.0 to 0.01 over 250,000 steps.

5. Atari Environment Preprocessing

Environment Wrapper Stack (DeepMind-style)

The Atari environment goes through a chain of wrappers that preprocess frames and modify the reward/termination signals. Applied in this exact order:

  1. gym.make(env_id) — Raw ALE environment
  2. NoopResetEnv(noop_max=30) — Random 1–30 NOOP actions on reset
  3. MaxAndSkipEnv(skip=4) — Repeat action 4×, return max of last 2 frames
  4. EpisodicLifeEnv — Treat life loss as episode end (training only)
  5. FireResetEnv — Press FIRE after reset (if game requires it)
  6. WarpFrame(84, 84) — RGB → grayscale, resize to 84×84
  7. ClipRewardEnv — Clip rewards to {−1, 0, +1} via np.sign (training only)
  8. FrameStack(k=4) — Stack last 4 frames → (4, 84, 84)
  9. UnionActionWrapper — Union → local action mapping (outermost)
Why this order matters: UnionActionWrapper must be the outermost wrapper so that inner wrappers like FireResetEnv still see the game’s native local action space. The agent issues union action indices, which the outermost wrapper maps to local indices before passing down through the wrapper chain. EpisodicLifeEnv and ClipRewardEnv are used only during training. Evaluation uses true game-over and raw unclipped rewards.

Observations

Shape: (4, 84, 84), dtype: uint8, range: [0, 255] (converted to [0, 1] float by the agent before the network forward pass).

Replay Buffer

Stores transitions as uint8 numpy arrays (memory-efficient): (state, action, reward, next_state, done) with state shape (4, 84, 84). Capacity: 200,000 transitions (default) or 1,000 (debug mode). Sampling: random uniform. Pixels normalized to [0, 1] float on sample.

6. DQN Agent and Training Loop

For each expert (one per Atari game), we run the standard DQN algorithm with the following components:

Double Network Architecture

  • Policy network — Updated every train_freq=4 environment steps
  • Target network — Hard-copied from policy every target_update_freq=2500 steps

Target Q-Value Computation

For a transition (s, a, r, s', done):

y = r + γ · maxa' ∈ valid Qtarget(s', a') · (1 − done)

where γ = 0.99 and the max is taken over valid actions only (masked).

Loss Function

Smooth L1 (Huber) loss between predicted and target Q-values:

L = SmoothL1(Qpolicy(s, a), y)

This is more robust to outliers than MSE.

Training Schedule

ParameterValue
Total timesteps1,000,000 per expert (5,000 debug)
Replay buffer min fill10,000 transitions

Epsilon decay1.0 → 0.01 linearly over 250K steps
Gradient clippingmax_norm = 10.0
OptimizerAdamW (lr = 1e-4)

Checkpointing

  • Best model (by evaluation reward) saved automatically as expert_{Game}_best.pt
  • Periodic step checkpoints are deleted when a new best is found (keeps only the best)
  • Each checkpoint contains: policy_net, target_net, optimizer, normalizer, step count

7. Q-Value Normalization

Different Atari games have vastly different reward scales:

GameReward Range
Pong{−1, 0, +1}
Breakout{0, 1, 4, 7} (per brick)
SpaceInvaders{0, 5, 10, 15, 20, 25, 30, …}

Without normalization, high-reward games would dominate the consolidation process.

Method: PopArt Normalization (Default)

“Preserving Outputs Precisely while Adaptively Rescaling Targets” (van Hasselt et al., 2016). Instead of normalizing Q-values externally, PopArt modifies the output layer weights to preserve the network’s predictions while adapting to changing target statistics:

μ(t) = (1 − m) · μ(t−1) + m · mean(targets)
σ(t) = (1 − m) · σ(t−1) + m · var(targets)

When the running statistics change, the output layer weights and biases are rescaled to preserve the network’s unnormalized outputs:

Wnew = Wold · σold / σnew
bnew = (σold · bold + μold − μnew) / σnew

Parameters: momentum m = 0.01.

Integration with Double DQN

During training, the target network’s outputs are denormalized before computing the Bellman target (r + γ max Qtarget). The resulting target is then normalized before computing the loss against the policy network’s output. This ensures the loss is computed in the normalized space while the Bellman backup operates in the original reward scale. Action selection via argmax is invariant to affine transforms.

Significance for consolidation: PopArt handles reward-scale differences between games naturally—no external normalization needed. In knowledge distillation, Q-values are additionally normalized per task before computing KL divergence.

8. Expert Initialization Strategy

Each expert is initialized independently with random weights. There is no shared global model or weight averaging between experts during training.

Procedure

  1. Expert 1 (Pong): Random init → Train on Pong for T steps → Save best checkpoint
  2. Expert 2 (Breakout): Random init → Train on Breakout for T steps → Save best checkpoint
  3. Expert 3 (SpaceInvaders): Random init → Train on SpaceInvaders for T steps → Save best checkpoint

Rationale

  • Independent initialization avoids coupling between the sequential training order—the order of games does not affect individual expert quality.
  • Each expert is free to find the best representation for its specific game without being biased by features from other games.
  • The consolidation methods (EWC, Distillation, HTCL) are specifically designed to merge independently trained experts. They do not require experts to start from a common initialization.
  • For HTCL, the initial global model for consolidation is simply the parameter average of all experts: wglobal = (1/T) · Σ wexpert_t. The Taylor expansion operates from this averaged starting point.
Note: Previous versions of this pipeline used global-to-local initialization (initializing each expert from a shared running average). This was removed because it coupled expert quality to training order and did not provide consistent benefits for the consolidation methods used here.

9. EWC Consolidation

Elastic Weight Consolidation (Kirkpatrick et al., 2017)

Concept

After training on a task, certain parameters are more “important” for that task. EWC uses the diagonal Fisher Information Matrix (FIM) as a proxy for importance. During consolidation, parameters with high Fisher values are penalized more for deviating from their task-optimal values.

Step 1: Compute Diagonal Fisher

For each task t with expert w*t and replay buffer Dt:

Fi(t) = Es~Dt [ (∂ log π(a|s) / ∂ θi)² ]

Approximated by sampling fisher_samples=2000 states from the replay buffer, computing the squared gradient of the log-policy (Q-values with softmax).

Step 2: Online EWC (Accumulate Across Tasks)

Fcumulative = γ · Fcumulative + Fnew

with γ = 0.95, giving exponentially decayed importance.

Step 3: Consolidation Objective

Starting from the parameter average of all experts, optimize:

L = LEWC + Lavg

where:

LEWC = (λEWC / 2T) · Σt Σi Fi(t) · (θi − θi*(t))²

Lavg = (1/T) · Σt || θ − θ*(t) ||²

λEWC = 5000. Optimization: AdamW for consolidation_steps=5000 (500 in debug).

Interpretation

  • Parameters important for a task (high Fisher) are pulled toward the task optimum.
  • Parameters unimportant for all tasks are free to move.
  • The parameter average term Lavg acts as a gentle pull toward all experts.
Limitations: Diagonal Fisher ignores parameter correlations. The quadratic penalty may be too conservative for large parameter shifts. Approximation quality degrades as the number of tasks grows.

10. Knowledge Distillation Consolidation

Concept

Train a “student” model (the global/consolidated model) to mimic the output distributions of multiple “teacher” models (the experts). Uses temperature-scaled softmax and KL divergence as the matching criterion.

Step 1: Collect Q-Value Statistics

For each expert/task, sample states from the replay buffer and compute μtask (mean) and σtask (std) of Q-values. These are used for per-task normalization.

Step 2: Distillation Loss (Per Task)

Given a batch of states from task t's replay buffer:

Q_teacher = frozen expert_t(states)  # shape: (batch, 6)
Q_student = global_model(states)     # shape: (batch, 6)

Normalize per task and compute soft targets:

pteacher = softmax(Q̂teacher / τ)     log qstudent = log_softmax(Q̂student / τ)

KL divergence with standard Hinton scaling (τ = 2.0):

Lt = τ² · KL(pteacher || qstudent)

Step 3: Training

For distill_epochs=50 (5 in debug), iterate over tasks in random order, sample batch from each task's replay buffer, compute distillation loss, and update the student with AdamW (lr = 5e-5).

Advantages Over EWC

  • Directly optimizes output similarity (behavioral cloning in Q-space).
  • Does not require Fisher computation.
  • Can capture richer information through the full Q-value distribution.

11. HTCL Consolidation (Taylor Expansion)

Hierarchical Taylor Consolidation Learning (Nag, Raghavan, Narayanan, 2026; arXiv:2602.02568)

Concept

Uses a second-order Taylor expansion of the loss function around the current global parameters. With a diagonal Fisher as the Hessian approximation, this yields a closed-form parameter update (no iterative optimization needed).

The Taylor Update

For each task t, with local expert weights wt and global model wglobal:

wglobalnew = wglobal + η · (H + λ I)−1 [λ · δd − g]

where:

  • δd = wt − wglobal (expert drift from global)
  • g = ∇J(wglobal) (gradient of cumulative past loss at global)
  • H = diag(F) (diagonal Fisher / Hessian approximation)
  • λ = regularization strength, η = 0.9 (step size)

Diagonal Simplification

Since H is diagonal, the inverse is trivial (element-wise):

(H + λ I)−1ii = 1 / (Hii + λ)

So the update for each parameter i is:

winew = wi + η · [λ · (wt,i − wi) − gi] / (Hii + λ)

Sequential Task Consolidation

Tasks are merged one at a time into the global model. The default task order places Pong last so it receives a corrective Taylor update:

  1. Initialize global from task 1 (Breakout) — register Fisher
  2. Merge task 2 (SpaceInvaders) into global → updated global
  3. Merge task 3 (Pong) into updated global → final global

Action-Masked Taylor Update

When the current expert uses fewer actions than the union space (e.g., Breakout uses 4 of 6 actions), the expert drift δd for head-layer rows corresponding to unused actions is zeroed out:

δd,i = 0 for action rows i ∉ valid_actions(expert)

This prevents untrained Q-head weights (e.g., Breakout’s RIGHTFIRE/LEFTFIRE rows, which contain random/untrained values) from corrupting the global model’s ability to distinguish those actions for games that do use them (Pong, SpaceInvaders).

Multi-Pass Consolidation

Rather than a single sequential pass, the task sequence is iterated num_passes times (default: 3). Each pass uses a geometrically decaying step size to prevent oscillation between expert optima:

ηpass = η × 0.3pass_idx   (η0 = 0.9, η1 = 0.27, η2 = 0.081)

At the start of each new pass (pass > 0), the cumulative Fisher and gradient are reset to zeros for a fresh Hessian estimate at the current model position. All tasks (including task-0) receive Taylor updates on passes > 0.

Joint Refinement

After multi-pass, a final refinement step:

  1. Computes Fisher and gradient jointly over all tasks at the final position
  2. Computes per-task Taylor corrections toward each expert under the joint Fisher
  3. Averages the correction vectors across all tasks
  4. Applies the averaged correction with a reduced step size (η × 0.1)
Compared to EWC: EWC uses an L2 penalty weighted by Fisher. HTCL uses the full second-order expansion, yielding an optimal direction, not just regularization. HTCL's closed-form update is a single step (no iterative optimization). HTCL explicitly accounts for the gradient at the global position, not just the distance to task optima.

12. Lambda Constraint and Per-Lambda Grid

Lambda Constraint

The Taylor update requires (H + λ I) to be positive definite:

λ > −μmin(H)

where μmin(H) is the minimum eigenvalue of the diagonal Hessian approximation. For a diagonal matrix, this equals the minimum diagonal entry: μmin = mini(Fi). Since Fisher values are always ≥ 0, we need λ > 0 (trivially satisfied for all default candidates).

Auto-Adjustment (lambda_auto=true)

λeffective = max(λ, −μmin + margin)

with margin = 0.1 (safety buffer). This guarantees (H + λI) is positive definite regardless of Fisher values.

Per-Lambda Consolidation

Rather than selecting a single λ via grid search, the consolidation pipeline runs independently for each λ candidate [0.1, 1.0, 10.0, 100.0, 1000.0]. Each candidate gets a fresh global model initialized from the first expert, producing separate consolidated checkpoints (consolidated_htcl_lam*.pt). The evaluation and comparison scripts discover all per-lambda checkpoints automatically and report results for each.

for lam in lambda_candidates:
    model = init_from_first_expert()
    for pass_idx in range(num_passes):
        eta_pass = eta * (0.3 ** pass_idx)
        for task in tasks:
            F_task = compute_fisher(model, task)
            g_task = compute_gradient(model, task)
            F_cum += F_task; g_cum += g_task
            model = taylor_update(model, expert_t, g_cum, F_cum,
                                  lam, eta_pass, valid_actions=task.actions)
    if joint_refinement:
        F_all, g_all = joint_fisher_gradient(model, all_tasks)
        corrections = [taylor_correction(expert_t, F_all, g_all, lam)
                       for expert_t in experts]
        model += mean(corrections) * (eta * 0.1)
    save(model, f"consolidated_htcl_lam{lam}.pt")
Effect: Multi-pass with decaying η avoids oscillation between experts. Action masking prevents Q-head corruption from experts with partial action spaces. Joint refinement provides a final balanced correction toward all experts simultaneously.

13. Evaluation and Comparison Pipeline

Evaluation Protocol

Each model (expert or consolidated) is evaluated on all tasks:

  1. Create evaluation environment: No episodic life (full game until all lives lost), no reward clipping (raw game rewards), same frame stacking and preprocessing otherwise.
  2. Run episodes: num_episodes (30 default, 2 debug) deterministic episodes. Action selection: argmax Qmasked(s, a) with no epsilon exploration. Safety limit: 27,000 steps per episode (~108K frames with skip=4).
  3. Report: mean, std, min, max, median reward.
