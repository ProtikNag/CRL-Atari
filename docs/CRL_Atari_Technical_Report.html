1. Overview and Motivation

This project implements a Continual Reinforcement Learning (CRL) framework for Atari games. The core challenge is training a single model that performs well across multiple Atari games without catastrophic forgetting, where learning a new task degrades performance on previously learned tasks.

Approach

The pipeline follows a train experts, then consolidate paradigm:

  1. Expert Training — Train separate DQN agents, one per Atari game (Pong, Breakout, SpaceInvaders). Each expert shares a unified 18-action output head covering the full Atari joystick action space, with game-specific action masking.
  2. Consolidation — Merge the expert models into a single global model using one of three methods:
    • Elastic Weight Consolidation (EWC) — Fisher-weighted parameter regularization
    • Knowledge Distillation — Temperature-scaled soft-target matching
    • HTCL — Second-order Taylor expansion with closed-form parameter updates
  3. Evaluation — Compare the consolidated models against individual experts on all tasks to measure knowledge retention and forward/backward transfer.

Key Design Decisions

  • Unified 18-action head — All experts output Q-values for all 18 Atari actions. Invalid actions are masked to −∞. This allows direct weight sharing and averaging.
  • Global-to-local initialization — Experts are initialized from a shared global model (initially random, then updated as the running average of trained experts).
  • Q-value normalization — Running statistics normalize Q-values to handle scale differences across games (Pong rewards in [−1, 1] vs. SpaceInvaders in [0, 500+]).
  • Diagonal Fisher — Used by both EWC and HTCL as a computationally tractable approximation to the Hessian, enabling element-wise closed-form updates in HTCL.
Reference: Nag, P., Raghavan, K., Narayanan, V. (2026). “Mitigating Task-Order Sensitivity and Forgetting via Hierarchical Second-Order Consolidation.” ICML 2026. arXiv:2602.02568.

3. DQN Network Architecture

The DQN uses a standard convolutional neural network (CNN) backbone followed by fully connected layers. The key architectural choice is the unified 18-action output head that covers the full Atari joystick action space across all games.

Architecture

Input:  (batch, 4, 84, 84)    # 4 stacked grayscale frames, 84×84 pixels

Layer 1: Conv2d(4 → 64,  kernel=8×8, stride=4, padding=0) + ReLU
         Output: (batch, 64, 20, 20)

Layer 2: Conv2d(64 → 128, kernel=4×4, stride=2, padding=0) + ReLU
         Output: (batch, 128, 9, 9)

Layer 3: Conv2d(128 → 128, kernel=3×3, stride=1, padding=0) + ReLU
         Output: (batch, 128, 7, 7)

Flatten: (batch, 128 × 7 × 7) = (batch, 6272)

FC:      Linear(6272 → 1024) + ReLU

Output:  Linear(1024 → 18)   # Q-values for all 18 actions

Total parameters: ~6.7 million

Dueling DQN (optional)

Instead of a single output head, splits into two streams:

  • Value stream: Linear(1024 → 512) → Linear(512 → 1)
  • Advantage stream: Linear(1024 → 512) → Linear(512 → 18)
Q(s, a) = V(s) + A(s, a) − meana' A(s, a')

Why 18 Actions?

The full Atari joystick has 18 possible actions (combinations of directions + fire). Each game uses a subset:

GameActions UsedIndices
Pong6 actions{0, 1, 3, 4, 11, 12}
Breakout4 actions{0, 1, 3, 4}
SpaceInvaders6 actions{0, 1, 3, 4, 11, 12}

By outputting all 18, we can directly average or merge expert weights without architectural mismatches. Invalid actions are masked to −∞.

4. Unified Action Space and Action Masking

The Arcade Learning Environment (ALE) defines 18 possible actions for the Atari joystick:

IndexActionIndexActionIndexAction
0NOOP6UPRIGHT12RIGHTFIRE
1FIRE7UPLEFT13UPRIGHTFIRE
2UP8DOWNRIGHT14UPLEFTFIRE
3RIGHT9DOWNLEFT15DOWNRIGHTFIRE
4LEFT10UPFIRE16DOWNLEFTFIRE
5DOWN11DOWNFIRE17UPLEFTFIRE

Action Masking

During action selection, the agent computes Q(s, a) for all 18 actions, then applies:

mask[a] = 0 if a ∈ valid_actions, −∞ otherwise

Qmasked(s, a) = Q(s, a) + mask[a]

a* = argmaxa Qmasked(s, a)

This ensures the agent never selects an invalid action while maintaining a shared weight structure.

Action Mapping (UnifiedActionWrapper)

The ALE environment expects local action indices (0 to n_game_actions − 1), not unified indices. The UnifiedActionWrapper translates:

Example for Pong (minimal actions = [0, 1, 3, 4, 11, 12]):
    unified action 0  → local index 0
    unified action 1  → local index 1
    unified action 3  → local index 2
    unified action 4  → local index 3
    unified action 11 → local index 4
    unified action 12 → local index 5

This wrapper is the innermost wrapper (applied right after gym.make) so all other wrappers see the standard local action space.

Epsilon-Greedy Exploration

During training, with probability ε, the agent selects a random action uniformly from the valid action set (not all 18 actions):

a ~ Uniform(valid_actions) with probability ε
a = argmax Qmasked(s, a) with probability 1 − ε

Epsilon decays linearly from 1.0 to 0.01 over 250,000 steps.

5. Atari Environment Preprocessing

Environment Wrapper Stack (DeepMind-style)

The Atari environment goes through a chain of wrappers that preprocess frames and modify the reward/termination signals. Applied in this exact order:

  1. gym.make(env_id) — Raw ALE environment
  2. UnifiedActionWrapper — Unified → local action mapping
  3. NoopResetEnv(noop_max=30) — Random 1–30 NOOP actions on reset
  4. MaxAndSkipEnv(skip=4) — Repeat action 4×, return max of last 2 frames
  5. EpisodicLifeEnv — Treat life loss as episode end (training only)
  6. FireResetEnv — Press FIRE after reset (if game requires it)
  7. ClipRewardEnv — Clip rewards to {−1, 0, +1} via np.sign
  8. WarpFrame(84, 84) — RGB → grayscale, resize to 84×84
  9. FrameStack(k=4) — Stack last 4 frames → (4, 84, 84)
Why this order matters: UnifiedActionWrapper must be innermost because it translates actions before any other wrapper processes them. EpisodicLifeEnv and ClipRewardEnv are used only during training. Evaluation uses true game-over and raw unclipped rewards.

Observations

Shape: (4, 84, 84), dtype: uint8, range: [0, 255] (converted to [0, 1] float by the agent before the network forward pass).

Replay Buffer

Stores transitions as uint8 numpy arrays (memory-efficient): (state, action, reward, next_state, done) with state shape (4, 84, 84). Capacity: 100,000 transitions (default) or 1,000 (debug mode). Sampling: random uniform. Pixels normalized to [0, 1] float on sample.

6. DQN Agent and Training Loop

For each expert (one per Atari game), we run the standard DQN algorithm with the following components:

Double Network Architecture

  • Policy network — Updated every train_freq=4 environment steps
  • Target network — Hard-copied from policy every target_update_freq=1000 steps

Target Q-Value Computation

For a transition (s, a, r, s', done):

y = r + γ · maxa' ∈ valid Qtarget(s', a') · (1 − done)

where γ = 0.99 and the max is taken over valid actions only (masked).

Loss Function

Smooth L1 (Huber) loss between predicted and target Q-values:

L = SmoothL1(Qpolicy(s, a), y)

This is more robust to outliers than MSE.

Training Schedule

OptimizerAdamW (lr = 1e-4)

Checkpointing

  • Best model (by evaluation reward) saved automatically
  • Periodic checkpoints every save_freq=50,000 steps
  • Final model saved at end of training
  • Each checkpoint contains: policy_net, target_net, optimizer, normalizer, step count

7. Q-Value Normalization

Different Atari games have vastly different reward scales:

GameReward Range
Pong{−1, 0, +1}
Breakout{0, 1, 4, 7} (per brick)
SpaceInvaders{0, 5, 10, 15, 20, 25, 30, …}

Without normalization, high-reward games would dominate the consolidation process.

Method 1: RunningNormalizer (Default)

Uses exponential moving average to track per-action running statistics:

μ(t) = (1 − m) · μ(t−1) + m · mean(Qbatch)
σ(t) = (1 − m) · σ(t−1) + m · var(Qbatch)
Q̂ = clamp( (Q − μ) / √(σ² + 10−8), −C, C )

Parameters: momentum m = 0.01, clip range C = 10.0.

Method 2: PopArtNormalizer (Optional)

“Preserving Outputs Precisely while Adaptively Rescaling Targets” (van Hasselt et al., 2016). Instead of normalizing Q-values externally, it modifies the output layer weights to preserve the network's predictions while changing the internal representation:

Wnew = Wold · σold / σnew
bnew = (σold · bold + μold − μnew) / σnew
Significance for consolidation: In knowledge distillation, we normalize Q-values per task before computing KL divergence. This ensures the distillation loss treats all games equally regardless of their raw Q-value scale.

8. Global-to-Local Initialization

A key component of the hierarchical training pipeline. Instead of initializing each expert from scratch (random weights), we initialize from a shared “global” model.

Procedure

  1. Create a random global model: wglobal ~ Random
  2. Expert 1 (Pong):
    • wPong := wglobal (initialize from global)
    • Train wPong on Pong for T steps
    • wglobal := wPong
  3. Expert 2 (Breakout):
    • wglobal := mean(wPong, wglobal)
    • wBreakout := wglobal
    • Train wBreakout on Breakout for T steps
    • wglobal := mean(wPong, wBreakout)
  4. Expert 3 (SpaceInvaders):
    • wglobal := mean of all prior experts + global
    • wSpaceInv := wglobal
    • Train wSpaceInv on SpaceInvaders for T steps

Why This Helps

  • Experts start from a shared representation, making their weights more “compatible” for later merging/consolidation.
  • Later experts benefit from features learned by earlier experts (positive transfer).
  • The unified output head means all 18 action Q-values get reasonable initialization.
  • This is particularly important for HTCL, where the closed-form Taylor update assumes expert weights are “close” to the global model. Starting from global ensures the drift δd = wexpert − wglobal is small.

9. EWC Consolidation

Elastic Weight Consolidation (Kirkpatrick et al., 2017)

Concept

After training on a task, certain parameters are more “important” for that task. EWC uses the diagonal Fisher Information Matrix (FIM) as a proxy for importance. During consolidation, parameters with high Fisher values are penalized more for deviating from their task-optimal values.

Step 1: Compute Diagonal Fisher

For each task t with expert w*t and replay buffer Dt:

Fi(t) = Es~Dt [ (∂ log π(a|s) / ∂ θi)² ]

Approximated by sampling fisher_samples=2000 states from the replay buffer, computing the squared gradient of the log-policy (Q-values with softmax).

Step 2: Online EWC (Accumulate Across Tasks)

Fcumulative = γ · Fcumulative + Fnew

with γ = 0.95, giving exponentially decayed importance.

Step 3: Consolidation Objective

Starting from the parameter-averaged global model, optimize:

L = LEWC + Lavg

where:

LEWC = (λEWC / 2T) · Σt Σi Fi(t) · (θi − θi*(t))²

Lavg = (1/T) · Σt || θ − θ*(t) ||²

λEWC = 5000. Optimization: AdamW for consolidation_steps=5000 (500 in debug).

Interpretation

  • Parameters important for a task (high Fisher) are pulled toward the task optimum.
  • Parameters unimportant for all tasks are free to move.
  • The parameter average term Lavg acts as a gentle pull toward all experts.
Limitations: Diagonal Fisher ignores parameter correlations. The quadratic penalty may be too conservative for large parameter shifts. Approximation quality degrades as the number of tasks grows.

10. Knowledge Distillation Consolidation

Concept

Train a “student” model (the global/consolidated model) to mimic the output distributions of multiple “teacher” models (the experts). Uses temperature-scaled softmax and KL divergence as the matching criterion.

Step 1: Collect Q-Value Statistics

For each expert/task, sample states from the replay buffer and compute μtask (mean) and σtask (std) of Q-values. These are used for per-task normalization.

Step 2: Distillation Loss (Per Task)

Given a batch of states from task t's replay buffer:

Q_teacher = frozen expert_t(states)  # shape: (batch, 18)
Q_student = global_model(states)     # shape: (batch, 18)

Normalize per task and compute soft targets:

pteacher = softmax(Q̂teacher / τ)     log qstudent = log_softmax(Q̂student / τ)

KL divergence with standard Hinton scaling (τ = 2.0):

Lt = τ² · KL(pteacher || qstudent)

Step 3: Training

For distill_epochs=50 (5 in debug), iterate over tasks in random order, sample batch from each task's replay buffer, compute distillation loss, and update the student with AdamW (lr = 5e-5).

Advantages Over EWC

  • Directly optimizes output similarity (behavioral cloning in Q-space).
  • Does not require Fisher computation.
  • Can capture richer information through the full Q-value distribution.

11. HTCL Consolidation (Taylor Expansion)

Hierarchical Taylor Consolidation Learning (Nag, Raghavan, Narayanan, 2026; arXiv:2602.02568)

Concept

Uses a second-order Taylor expansion of the loss function around the current global parameters. With a diagonal Fisher as the Hessian approximation, this yields a closed-form parameter update (no iterative optimization needed).

The Taylor Update

For each task t, with local expert weights wt and global model wglobal:

wglobalnew = wglobal + η · (H + λ I)−1 [λ · δd − g]

where:

  • δd = wt − wglobal (expert drift from global)
  • g = ∇J(wglobal) (gradient of cumulative past loss at global)
  • H = diag(F) (diagonal Fisher / Hessian approximation)
  • λ = regularization strength, η = 0.9 (step size)

Diagonal Simplification

Since H is diagonal, the inverse is trivial (element-wise):

(H + λ I)−1ii = 1 / (Hii + λ)

So the update for each parameter i is:

winew = wi + η · [λ · (wt,i − wi) − gi] / (Hii + λ)

Sequential Task Consolidation

Tasks are merged one at a time into the global model:

  1. Merge task 1 (Pong) into global → updated global
  2. Merge task 2 (Breakout) into updated global → updated global
  3. Merge task 3 (SpaceInvaders) → final global

After each merge, catch-up iterations refine the result (see next section).

Compared to EWC: EWC uses an L2 penalty weighted by Fisher. HTCL uses the full second-order expansion, yielding an optimal direction, not just regularization. HTCL's closed-form update is a single step (no iterative optimization). HTCL explicitly accounts for the gradient at the global position, not just the distance to task optima.

12. Lambda Constraint and Catch-Up Iterations

Lambda Constraint

The Taylor update requires (H + λ I) to be positive definite:

λ > −μmin(H)

where μmin(H) is the minimum eigenvalue of the diagonal Hessian approximation. For a diagonal matrix, this equals the minimum diagonal entry: μmin = mini(Fi). Since Fisher values are always ≥ 0, we need λ > 0 (trivially satisfied for default λ = 1.0).

Auto-Adjustment (lambda_auto=true)

λeffective = max(λ, −μmin + margin)

with margin = 0.1 (safety buffer). This guarantees (H + λI) is positive definite regardless of Fisher values.

Catch-Up Iterations

After the initial Taylor update for task t, we perform catch_up_iterations=2 refinement steps:

  1. Recompute the gradient g at the new global position (the cumulative Fisher stays the same)
  2. Apply the Taylor update again with the new gradient

This is a form of fixed-point iteration: the initial update uses the gradient at the old position; catch-up iterations “catch up” to the gradient at the new position. Converges quickly because the Fisher (curvature) changes slowly.

for task_t in tasks:
    w_global = taylor_update(w_global, w_expert_t, g, F_cum)
    for k in range(catch_up_iters):
        g_new = compute_gradient(w_global)     # gradient at new position
        w_global = taylor_update(w_global, w_expert_t, g_new, F_cum)
    F_cum += F_task_t
    g_cum += g_new
Effect: Without catch-up, we get a single Newton-like step that may overshoot. With 2 catch-up iterations, we achieve significantly better convergence in practice, as the gradient is re-evaluated at the updated position.

13. Evaluation and Comparison Pipeline

Evaluation Protocol

Each model (expert or consolidated) is evaluated on all tasks:

  1. Create evaluation environment: No episodic life (full game until all lives lost), no reward clipping (raw game rewards), same frame stacking and preprocessing otherwise.
  2. Run episodes: num_episodes (30 default, 2 debug) deterministic episodes. Action selection: argmax Qmasked(s, a) with no epsilon exploration. Safety limit: 27,000 steps per episode (~108K frames with skip=4).
  3. Report: mean, std, min, max, median reward.
